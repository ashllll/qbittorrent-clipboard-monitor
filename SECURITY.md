## 安全特性

### 1. 敏感信息过滤

项目实现了全面的敏感信息过滤机制：

- **自动日志过滤**：自动遮盖密码、API密钥、令牌等敏感信息
- **磁力链接脱敏**：隐藏磁力链接hash的后32位，保留前8位用于识别
- **配置脱敏**：配置文件中敏感字段自动遮盖

### 2. 输入验证

所有外部输入都经过严格验证：

- **磁力链接验证**：
  - 检查协议格式（`magnet:?`）
  - 验证btih hash长度（32位或40位）
  - 检查非法字符和控制字符
  - 限制最大长度（8192字符）防止DoS

- **路径验证**：
  - 防止路径遍历攻击（`../`、`..\`等）
  - 检查非法字符（`<`、`>`、`:`、`"`、`|`、`?`、`*`）
  - 检查Windows保留名称（CON、PRN、AUX等）
  - 阻止环境变量扩展（`$VAR`）

- **URL验证**：
  - 仅允许HTTP/HTTPS协议
  - 禁止URL中包含认证信息（`user:pass@host`）
  - 验证URL格式正确性

- **主机名验证**：
  - 检查命令注入风险字符
  - 验证主机名长度（最大253字符）

### 3. 安全配置

- **文件权限**：配置文件自动设置为 0o600（仅用户可读写）
- **超时设置**：所有网络请求都有合理的超时限制
- **重试机制**：指数退避重试，防止资源耗尽
- **速率限制**：限制每次检查处理的磁力链接数量（默认100个）

### 4. HTTP安全

- **安全头部**：所有HTTP请求包含标准的User-Agent和Accept头部
- **连接池限制**：限制连接数防止资源耗尽
- **SSL/TLS**：支持HTTPS连接（推荐用于生产环境）

## 报告安全漏洞

如果您发现了安全漏洞，请通过以下方式报告：

1. **不要**在公共Issue中披露漏洞细节
2. 发送邮件至：`security@qb-monitor.local`
3. 提供以下信息：
   - 漏洞描述
   - 复现步骤
   - 受影响的版本
   - 可能的修复建议

我们承诺：
- 在24小时内确认收到报告
- 在7天内提供初步评估
- 在修复后公开致谢（如果您希望）

## 安全最佳实践

### 配置安全

1. **使用环境变量存储敏感信息**：
   ```bash
   export QBIT_PASSWORD="your-secure-password"
   export AI_API_KEY="your-api-key"
   ```

2. **设置安全的文件权限**：
   ```bash
   chmod 600 ~/.config/qb-monitor/config.json
   ```

3. **使用HTTPS**：
   ```json
   {
     "qbittorrent": {
       "use_https": true
     }
   }
   ```

### 运行安全

1. **定期更新依赖**：
   ```bash
   pip install --upgrade -r requirements.txt
   ```

2. **监控日志**：定期检查日志文件是否有异常活动

3. **限制网络访问**：确保qBittorrent Web UI仅在可信网络中可访问

### Docker安全

1. **使用非root用户运行**：
   ```dockerfile
   USER 1000:1000
   ```

2. **限制容器权限**：
   ```yaml
   security_opt:
     - no-new-privileges:true
   read_only: true
   ```

3. **使用只读挂载**：
   ```yaml
   volumes:
     - ./config:/app/config:ro
   ```

## 依赖安全

我们使用以下工具监控依赖安全：

- `pip-audit`：扫描Python依赖中的已知漏洞
- `safety`：检查依赖包的安全问题

建议定期运行：
```bash
pip install pip-audit
pip-audit --desc --requirement requirements.txt
```

## 安全相关配置选项

### qbittorrent

| 选项 | 说明 | 安全建议 |
|------|------|----------|
| `host` | 服务器地址 | 使用本地地址或受限网络 |
| `port` | 服务端口 | 避免使用默认8080端口 |
| `username` | 用户名 | 使用强密码 |
| `password` | 密码 | 通过环境变量设置，避免硬编码 |
| `use_https` | 使用HTTPS | 生产环境必须启用 |

### AI配置

| 选项 | 说明 | 安全建议 |
|------|------|----------|
| `api_key` | API密钥 | 通过环境变量设置，定期轮换 |
| `base_url` | API基础URL | 验证URL合法性 |

### 监控配置

| 选项 | 说明 | 安全建议 |
|------|------|----------|
| `check_interval` | 检查间隔 | 合理设置，避免过于频繁的检查 |
| `log_level` | 日志级别 | 生产环境使用INFO或更高级别 |
